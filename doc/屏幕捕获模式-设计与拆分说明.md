# 屏幕捕获模式设计与实现说明

## 1. 文档目的

本文档说明当前“屏幕捕获模式”的实现架构、默认后端选择、能力边界与构建开关，作为后续维护与扩展（窗口捕获、区域捕获、录制）的依据。

---

## 2. 当前结论（2026-04）

- 屏幕采集能力统一走 `IScreenCapture` 抽象。
- Windows 默认屏幕采集后端为 **DXGI Desktop Duplication**（`MediaBackendType::Dxgi`）。
- FFmpeg `gdigrab` 保留为可回退后端（或在非 DXGI 条件下使用）。
- UI 仅做交互展示，采集逻辑全部在 backend/service/runtime 链路。

---

## 3. 分层设计

### 3.1 API 层

- `api/include/fplayer/api/media/iscreencapture.h`
- `api/src/iscreencapture.cpp`

核心定义：

- `ScreenDescription`：屏幕元数据（名称、主副屏、像素分辨率、坐标）
- `IScreenCapture`：统一接口（枚举/选择/绑定预览/启停/鼠标/帧率）

### 3.2 后端实现层

- Qt 实现
  - `backend/media_qt6/include/fplayer/backend/media_qt6/screencaptureqt6.h`
  - `backend/media_qt6/src/screencaptureqt6.cpp`

- FFmpeg 实现（gdigrab）
  - `backend/media_ffmpeg/include/fplayer/backend/media_ffmpeg/screencaptureffmpeg.h`
  - `backend/media_ffmpeg/src/screencaptureffmpeg.cpp`

- DXGI 实现（Desktop Duplication）
  - `backend/media_dxgi/include/fplayer/backend/media_dxgi/screencapturedxgi.h`
  - `backend/media_dxgi/src/screencapturedxgi.cpp`
  - `backend/media_dxgi/CMakeLists.txt`

### 3.3 Runtime 层

- `runtime/src/runtime.cpp` 按 `MediaBackendType` 创建具体实现。
- `MediaBackendType::Dxgi` 在 Windows + 启用 DXGI 模块时返回 `ScreenCaptureDxgi`。
- 若未构建 DXGI 模块，`Dxgi` 分支回退到 `ScreenCaptureFFmpeg`。

### 3.4 Service 层

- `service/include/fplayer/service/service.h`
- `service/src/service.cpp`

提供统一业务入口：

- `initScreenCapture(...)`
- `bindScreenPreview(...)`
- `getScreenList()` / `selectScreen(...)`
- `screenSetActive(...)` / `screenIsActive()`
- `screenSetCursorCaptureEnabled(...)` / `screenCanControlCursorCapture()`
- `screenSetFrameRate(...)` / `screenCanControlFrameRate()`

### 3.5 Widget 层

- `widget/src/capturewindow.cpp`

职责：

- 模式切换
- 控件显隐与状态展示
- 调用 Service，不直接接触后端细节

## 4. 运行时流程（屏幕模式）

### 4.1 进入屏幕模式

1. `CaptureWindow` 切换到 `Screen`
2. 选择屏幕采集后端（Windows 默认 DXGI）
3. `Service::bindScreenPreview()` 绑定 `FVideoView`
4. `Service::getScreenList()` 刷新下拉
5. `Service::selectScreen(index)` 选择目标屏幕
6. `Service::screenSetActive(true)` 开始采集

### 4.2 切屏

1. 下拉变化触发 `selectScreen(index)`
2. 后端停止当前采集并重建对应输出/输入对象
3. 持续输出到同一预览控件

### 4.3 播放/暂停按钮映射

- 屏幕模式下映射到 `screenSetActive(true/false)`。

## 5. DXGI 后端实现说明

### 5.1 核心流程

`ScreenCaptureDxgi` 采用：

- `IDXGIOutputDuplication::AcquireNextFrame()` 获取桌面帧
- D3D11 staging texture 映射到 CPU
- BGRA 数据通过 `libswscale` 转为 `YUV420P`
- `QMetaObject::invokeMethod(..., Qt::QueuedConnection)` 投递给 `FGLWidget::updateYUVFrame`

### 5.2 多显示器识别

- 屏幕列表来源于 Qt `QGuiApplication::screens()`，并统一为像素坐标（考虑 DPR）。
- DXGI 输出枚举时用 `DXGI_OUTPUT_DESC::DesktopCoordinates`（left/top/width/height）匹配 `ScreenDescription`，实现屏幕索引映射。

### 5.3 帧率支持

- 支持 `setFrameRate(int fps)` 动态设置采样节奏（循环内按 `1000/fps` 控制）。
- UI 侧沿用动态帧率下拉与推荐策略。

### 5.4 鼠标指针支持

- 支持开关控制是否在帧上叠加鼠标指针。
- 叠加方式为 `GetCursorInfo` + `DrawIconEx` 绘制到 BGRA 帧后再转 YUV。
- 相比 `gdigrab + CAPTUREBLT` 路径，通常可显著减少“系统光标全局闪烁”。

## 6. 构建与启用

### 6.1 CMake 开关

文件：`backend/CMakeLists.txt`

- `FPLAYER_BUILD_MEDIA_DXGI`（默认 `ON`，且仅 `WIN32` 时生效）

### 6.2 关联模块

- `backend/media_dxgi/CMakeLists.txt`
  - 链接 `d3d11`, `dxgi`
  - 链接 `FPlayer_Common`（用于 `FGLWidget`）
  - 链接 `swscale`, `avutil`

- `runtime/CMakeLists.txt`
  - 启用 DXGI 时链接 `FPlayer_Backend_Desktop_Capture_Dxgi`
  - 导出编译宏 `FPLAYER_WITH_SCREEN_DXGI=1`

### 6.3 默认行为

- `CaptureWindow` 在 `_WIN32 + FPLAYER_WITH_SCREEN_DXGI` 条件下默认选择 `MediaBackendType::Dxgi`。
- 否则回退 `MediaBackendType::FFmpeg`。

---

## 7. 已知限制

- 当前仅支持“按屏幕捕获”，未提供“窗口捕获/区域捕获”。
- DXGI 在部分场景（会话切换、显卡切换、权限/桌面状态变化）可能出现 duplication 丢失，当前实现会尝试重建，但仍建议补充更细错误码与 UI 提示。
- 目前路径为 CPU 映射 + swscale 转换，后续可优化为 GPU 路径降低 CPU 占用。
