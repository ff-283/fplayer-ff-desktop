# 推拉流使用说明

## 1. 功能入口

在主窗口顶部菜单栏中，点击：

- `推拉流` -> `推流`
- `推拉流` -> `拉流`

会弹出对应的配置窗口。

---

## 2. 推流使用

## 2.1 配置项说明

- `输入`：推流输入源（本地文件路径或可被 ffmpeg 识别的输入地址）
- `协议模板`：快速填充输出地址（RTMP / RTSP / SRT）
- `输出`：推送目标地址

常见示例：

- RTMP：`rtmp://127.0.0.1:1935/live/stream`
- RTSP：`rtsp://127.0.0.1:8554/live/stream`
- SRT：`srt://127.0.0.1:8890?mode=caller`

## 2.2 操作流程

1. 点击 `推拉流` -> `推流`
2. 选择输入源（可用“选择文件”按钮）
3. 选择协议模板（可选）
4. 填写输出地址
5. 点击 `确定` 启动推流
6. 如需停止，点击窗口内 `停止推流`

## 2.3 状态与日志

窗口内提供：

- `状态`：运行中 / 已停止 + 退出码
- `日志`：ffmpeg 最近输出（自动刷新）

---

## 3. 拉流使用

## 3.1 配置项说明

- `输入`：源流地址
- `协议模板`：快速填充输入地址（RTMP / RTSP / SRT）
- `输出`：输出文件路径（不勾选预览时生效）
- `直接预览到播放窗口`：勾选后不落地文件，直接在主播放窗口预览

常见示例：

- 输入流：`rtmp://127.0.0.1:1935/live/stream`
- 输出文件：`D:/temp/pull.mp4`

## 3.2 操作流程（落地文件）

1. 点击 `推拉流` -> `拉流`
2. 填写输入流地址
3. 填写输出文件（可用“选择输出文件”）
4. 不勾选“直接预览到播放窗口”
5. 点击 `确定` 开始拉流
6. 如需停止，点击 `停止拉流`

## 3.3 操作流程（直接预览）

1. 点击 `推拉流` -> `拉流`
2. 填写输入流地址
3. 勾选 `直接预览到播放窗口`
4. 点击 `确定`
5. 主窗口切换到播放器预览逻辑，直接显示流内容

---

## 4. 历史记录

推流和拉流都支持最近地址记录（会话内）：

- 最近推流输入/输出
- 最近拉流输入/输出

每项最多保留 8 条，最新项在最前面。

---

## 5. 常见问题

## 5.1 点击确定后提示启动失败

可能原因：

- 本机找不到 `ffmpeg.exe`
- 输入地址不可达
- 输出地址不可写/不可连

建议：

- 确保程序目录有 `ffmpeg.exe`，或系统 `PATH` 可找到 `ffmpeg`
- 检查地址、端口、防火墙

## 5.2 推流成功但播放器看不到画面

建议排查：

- 推流协议与播放端协议是否一致
- 媒体服务器是否正常（如 rtmp/rtsp 服务）
- 目标 URL 路径是否正确

## 5.3 拉流预览无画面

建议排查：

- 输入流是否可用（先用 ffplay/vlc 验证）
- 网络是否连通
- 主窗口当前是否被其他模式占用（重复切换模式后重试）
