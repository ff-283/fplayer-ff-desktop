# 推拉流开发文档

## 1. 目标与范围

本文档描述当前“推拉流”功能的工程落地方式，包括：

- 分层架构
- 关键接口
- UI 交互流程
- FFmpeg 执行策略
- 已知限制与后续扩展建议

当前版本聚焦“可用闭环”，以 FFmpeg 进程方式实现推流与拉流。

---

## 2. 分层架构

## 2.1 API 层

文件：

- `api/include/fplayer/api/net/istream.h`
- `api/src/istream.cpp`

核心抽象：`fplayer::IStream`

主要接口：

- `startPush(inputUrl, outputUrl)`
- `startPull(inputUrl, outputUrl)`
- `stop()`
- `isRunning()`
- `lastError()`
- `recentLog()`
- `lastExitCode()`

---

## 2.2 后端实现层（FFmpeg）

文件：

- `backend/net_ffmpeg/include/fplayer/backend/net_ffmpeg/streamffmpeg.h`
- `backend/net_ffmpeg/src/streamffmpeg.cpp`
- `backend/net_ffmpeg/CMakeLists.txt`

实现类：`fplayer::StreamFFmpeg`

实现策略：

- 使用 `QProcess` 启动 ffmpeg 子进程
- 合并 stdout/stderr 作为日志源
- 缓存最近日志（截断上限避免无限增长）
- 记录最近退出码

命令策略（当前）：

- 推流：`-re -i <input> -c copy -f flv <output>`
- 拉流：`-i <input> -c copy -y <output>`

说明：当前默认采用转封装（`-c copy`）以降低 CPU 消耗并快速打通。

---

## 2.3 Runtime 层

文件：

- `runtime/include/fplayer/runtime/runtime.h`
- `runtime/src/runtime.cpp`
- `runtime/CMakeLists.txt`

新增：

- `createStream(MediaBackendType backend)`

当后端类型为 `FFmpeg` 时，返回 `StreamFFmpeg` 实例。

---

## 2.4 Service 层

文件：

- `service/include/fplayer/service/service.h`
- `service/src/service.cpp`

新增能力：

- 初始化：`initStream(...)`
- 控制：`streamStartPush(...)` / `streamStartPull(...)` / `streamStop()`
- 状态：`streamIsRunning()` / `streamLastError()`
- 日志：`streamRecentLog()` / `streamLastExitCode()`

Service 作为 UI 与后端之间的统一入口，避免 UI 直接触碰后端对象。

---

## 2.5 Widget/UI 层

文件：

- `widget/src/capturewindow.cpp`

新增入口：

- 顶部菜单 `推拉流`（与 `模式` 并列）
  - `推流`
  - `拉流`

新增交互：

- 推流配置对话框
- 拉流配置对话框
- 协议模板（RTMP / RTSP / SRT）
- 最近地址下拉（会话内）
- 文件选择按钮
- 实时状态与日志显示
- 拉流“直接预览到播放窗口”模式

## 3. 关键流程

## 3.1 推流流程

1. 用户打开 `推流` 对话框
2. 配置输入/输出地址
3. 调用 `Service::streamStartPush(...)`
4. Service 转发到 `IStream::startPush(...)`
5. `StreamFFmpeg` 组装命令并启动 `QProcess`
6. UI 定时拉取状态与日志展示

## 3.2 拉流流程（文件落地）

1. 用户打开 `拉流` 对话框
2. 配置输入流与输出文件
3. 调用 `Service::streamStartPull(...)`
4. FFmpeg 拉流并写入目标文件

## 3.3 拉流流程（直接预览）

1. 用户勾选 `直接预览到播放窗口`
2. 不走 `streamStartPull` 落盘流程
3. 直接调用现有播放器链路 `openMediaFile(inputUrl)` 进入播放预览

## 4. CMake 与模块开关

文件：`backend/CMakeLists.txt`

- `FPLAYER_BUILD_NET_FFMPEG` 当前默认设为 `ON`
- `backend/net_ffmpeg` 会被纳入构建

文件：`runtime/CMakeLists.txt`

- 启用 `FPLAYER_BUILD_NET_FFMPEG` 时链接 `FPlayer_Backend_Net_FFmpeg`

---

## 5. 设计取舍

## 5.1 为什么用 QProcess 而不是直接嵌入 libav*

- 落地速度快，容易验证
- 命令与日志可直接对照 ffmpeg CLI，调试成本低
- 与现有项目结构耦合较低

## 5.2 为什么拉流预览复用播放器链路

- 避免重复实现网络解复用与渲染路径
- 复用现有 `player` 预览能力，集成成本最低

---

## 6. 已知限制

- 推流输入源目前主要面向文件/URL，设备采集参数尚未做结构化配置
- 推流协议输出当前统一使用 `-f flv`，对 RTSP/SRT 的协议细化仍可继续增强
- 地址历史仅会话内保留，未做持久化
- 日志是“最近片段”，不是完整历史

---

## 7. 后续扩展建议

- 增加协议感知的参数模板：
  - RTMP/RTSP/SRT 分别使用更合理的容器与 mux 参数
- 增加推流高级参数：
  - 编码器、码率、GOP、B 帧、音频参数
- 增加任务并发管理：
  - 支持多任务并行推/拉流
- 地址历史持久化（配置文件）
- 增加标准错误码映射，便于 UI 给出更清晰提示
